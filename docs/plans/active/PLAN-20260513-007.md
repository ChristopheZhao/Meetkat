## 2. Architecture Direction
- Supervisor = conductor. Emits `SpeakerSlot[]` (`{agent, run, order, focus_angle?}`) and `decision_focus`. No content fields.
- Sub-agents = autonomous authors. Each specialist receives: decision_focus, role contract, optional focus_angle hint, memory_recall (shared facts + per-role long-term lessons + recent room events), recent transcript. Specialist decides its claim, evidence, and confidence.
- Memory:
  - Short-term: `RoomMemoryStore` keyed by `(room_id, scope)` where scope is `mas:{room_id}` (shared) or `agent:{room_id}:{role}` (per-agent). Facts dict + event deque.
  - Long-term: `LongTermLessonStore` keyed by role. Append-only JSON of {ts, room_id, lesson_text, decision_record_ref}. Initially populated by synthesis writeback at meeting end; read by specialist prompts as "prior lessons in your remit".
  - Storage: file under `${MEETKAT_MEMORY_DIR:-~/.meetkat/memory}/`. Override path with env var for tests.
  - SoT discipline: every memory.write also publishes a `memory.write` room event so the journal owns the truth; the file is a denormalized projection.
- Frontend:
  - All product copy → Chinese. Keep English in code comments / dev tools.
  - Visual: gradient hero + frosted card surface + role color accents (decision-scribe = amber, systems-architect = teal, etc.) + animated typing dots for in-flight specialist turns.
