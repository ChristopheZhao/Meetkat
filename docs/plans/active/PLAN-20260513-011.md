## 2. Execution Sprints

### Sprint E1: 决策记录导出器 + 端到端 case
- `decision_record.render_decision_record(snapshot, *, room_memory_snapshot, role_lessons)` 是 pure projection，无副作用、无 provider 调用。Sections：
  - 元数据 / 原始需求 / 会议目标
  - 决策结论（candidate + reason）
  - 行动项 markdown checklist
  - 待办开放问题
  - 各角色立场摘要（每个 specialist 最后一条 claim + evidence + confidence）
  - 主持人选角与焦点（逐轮 supervisor.plan 事件 + latest_supervisor_plan fact）
  - 角色长期 Lesson 摘要（每个 role 最近 3 条）
  - 完整对话记录（agent.message / human.message / human.override）
- `scripts/run_meeting_and_export.py` 用 `centralized` executor 跑多轮会议、轮询到 ended、调用渲染器、写入 `docs/decisions/{YYYY-MM-DD}-{topic-slug}.md`。
- 实跑 case：18 人团队按业务能力拆 2-3 个微服务（边界、节奏、IPC、数据/事件、运维可观测性）。

### Sprint E2: image-gen avatar 重生成路径占位
- `frontend/src/assets/agents/README.md` 记录：
  - 5 张 avatar 文件名 / role 映射 / 显示名表
  - 共享视觉契约（光照、配色、构图、风格）
  - per-role prompt 模板
  - 文件系统约定：新角色需同时 (a) 加入 `_ROLE_BLUEPRINTS`、(b) 添加 avatar、(c) 注册到 `roleAvatarMap`
- `docs/decisions/README.md` 记录决策文档生成路径 + 各 section 含义 + 可重现性保证（journal replay）。
