## 1. Requirement Analysis
- Trigger: post-delivery adversarial audit of PLAN-007 found 4 architecture gaps. User asked "检查下当前的实现是否符合架构设计，边界原则了？" and prompted continuation with the 4 fixes outlined below.
- Findings recap:
  - **CRITICAL** (baseline §2.5 + §2.9 violation): PLAN-007 §1 promised "Memory writes are journal-anchored: every memory.write produces a memory.write room event so RoomEventJournal stays single SoT and the memory store is a projection target". Implementation did not do this — `RoomMemoryStore.write_fact` was called directly from `central_executor._record_specialist_memory`, bypassing the journal entirely. The store became a second source of truth that could not be rebuilt by replaying the journal.
  - **MODERATE**: `persist_meeting_lessons_from_snapshot` wrote the same decision-record text to every participating role's long-term lesson file. Each role's file had identical `.text`, so "per-role learning" was a misnomer — the system was broadcasting the decision record into N files.
  - **MODERATE**: Supervisor's `decision_focus / reason / open_questions / speakers` were never written to memory, so the next round's supervisor's `memory_recall` could not see what the previous supervisor decided.
  - **MINOR**: `focus_angle` allowed by the supervisor prompt could become a content-prescription backdoor — there was no parse-time guard against the supervisor writing "argue X side" or "建议采用 Y".
- Scope in:
  - B1: route every per-round memory write through a `memory.write` journal event; projector dispatches to the store; cold replay of the journal rebuilds the store.
  - B2: per-role lesson text anchored to that role's most-recent transcript claim.
  - B3: supervisor self-memory writeback at start of each round (`latest_supervisor_plan` fact + `supervisor.plan` event in mas scope).
  - B4: `sanitize_focus_angle` strips supervisor focus_angle hints that contain prescriptive verbs (en + zh).
- Scope out:
  - LLM-generated per-role lessons (synthesis-time LLM call to write each role's "what I learned" line). Deferred — current "role's own claim + meeting candidate" heuristic is sufficient for cross-room recall ranking.
  - Frontend memory inspector. Deferred.
  - Replacing `HybridConsensusStrategy` / phase-rules / signal increments with LLM-judged equivalents. Documented as long-running SHAPING debt, not BLOCKER.
- Acceptance criteria:
  - Every memory write in the centralized MAS path produces a `memory.write` journal event (verified by smoke + unit test).
  - `RoomMemoryStore.apply_journal_event` rebuilds state from journal-shaped payloads (verified by replay test).
  - Per-role lesson text differs across roles after a meeting closes (verified by smoke).
  - `latest_supervisor_plan` fact + `supervisor.plan` event present in mas scope after each round.
  - `sanitize_focus_angle` strips English (argue/claim/conclude/recommend/propose/advocate/insist/demand) and Chinese (主张/论证/得出结论/结论是/建议/提议/坚持/要求/断言/应该认为/应该说/应该主张) prescriptive verbs.
  - Backend + frontend test sweeps green; real-LLM smoke green.
- Architecture constraints:
  - Must respect `MAS-harness-architecture-baseline.md` §2.5 (RoomEventJournal single SoT) and §2.9 (no second fact chain).
  - Standalone executor path (without a runtime / publish callable) must still work for unit tests — fall back to direct store writes there, documented as test-only mode.
  - No frontend changes required in this sprint — `central_mas` artifact bundle shape unchanged.

## 2. Execution Sprints

### Sprint B1: journal-anchored memory writes
- `RoomMemoryStore.apply_journal_event(payload, room_id)` parses a journal-shaped `memory.write` payload (`{scope, fact_key?, fact_value?, memory_event_type?, memory_event_payload?}`) and updates local state via existing `write_fact` / `record_event` methods.
- `RoomProjector.__init__(room_id, memory_store=None)` accepts a memory store. `apply()` adds a `memory.write` case that routes to `store.apply_journal_event(event.payload, room_id)` and swallows store errors so memory projection cannot poison snapshot projection.
- `RoomRuntime.create_room` injects `self._room_memory` into the projector at construction.
- `RoomRuntime._drive_room` passes `publish=self.publish` into `self._orchestrator.build_round`.
- `RoomOrchestrator.build_round(snapshot, round_index, *, publish=None)` forwards the publish callable to the executor; catches `TypeError` so executors that have not adopted the new signature stay compatible.
- `CentralizedMASExecutor.build_round` accepts `publish` and threads it through `_record_supervisor_plan` and `_record_specialist_memory`.
- `CentralizedMASExecutor._emit_memory_write` is the single seam: when `publish` is provided, it publishes a `memory.write` event (journal anchors the write); when `publish` is `None` (test/standalone mode), it falls back to direct store writes.

### Sprint B2: per-role lessons
- `persist_meeting_lessons_from_snapshot` walks the transcript and builds a `role_claims` map of role → last-claim text + confidence.
- Per-role lesson text now: `"作为 {role}，我主张：{their last claim}（信心 {x.xx}）。本会议候选决策：{candidate}"`. Long-term `role_lessons` recall now represents per-role learning instead of broadcast decision records.
- Fallback: if a role participated but no parseable claim is in the transcript, the legacy broadcast text is used.

### Sprint B3: supervisor self-memory
- `_record_supervisor_plan` runs immediately after the supervisor LLM call, before specialists generate their arguments.
- Writes `latest_supervisor_plan` fact (`{round_index, phase, decision_focus, current_focus, reason, open_questions, speakers[]}`) to mas scope + records a `supervisor.plan` event with the runnable speaker list. Both go through `_emit_memory_write` so they are journal-anchored too.

### Sprint B4: focus_angle prescription guard
- New `sanitize_focus_angle(value)` exported from `central_mas` and `orchestration` package.
- Two-tier detection: English `\b(argue|claim|conclude|recommend|propose|advocate|insist|demand)\b` regex + Chinese substring match against (`主张` / `论证` / `得出结论` / `结论是` / `建议` / `提议` / `坚持` / `要求` / `断言` / `应该认为` / `应该说` / `应该主张`). (Word boundary `\b` does not work for CJK so substring is the correct primitive.)
- `parse_supervisor_plan` runs every speaker's raw focus_angle through `sanitize_focus_angle`. Stripped values become `""` (no hint), preserving the supervisor's selection while neutralizing content prescription.
