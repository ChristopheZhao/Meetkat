# Decision Records

Generated by `scripts/run_meeting_and_export.py` after a real-LLM
centralized MAS meeting closes. Each file is a denormalized projection of:

- `RoomEventJournal` (single source of truth) — the meeting's event stream
- `RoomMemoryStore` snapshot — shared facts + supervisor.plan events per round
- `LongTermLessonStore` — per-role lessons accumulated across rooms

If a file looks out of date or missing, the underlying snapshot is still
authoritative; re-run the script with the same `--requirement` to
regenerate, or render directly via
`decision_room.runtime.decision_record.render_decision_record`.

## Sections produced

1. **元数据** — room id, status, round count, decision type, generation time
2. **原始需求 / 会议目标** — operator-supplied requirement + LLM planner goal
3. **决策结论** — `candidate_decision` + `conclusion_reason`
4. **行动项** — `action_items` rendered as a markdown checklist
5. **待办的开放问题** — unresolved questions the agents flagged
6. **各角色立场摘要** — each specialist's most-recent claim + evidence + confidence
7. **主持人选角与焦点（按轮次）** — supervisor.plan events with focus_angle hints
8. **角色长期 Lesson 摘要** — what each role wrote to its long-term store at meeting end
9. **完整对话记录** — full transcript of `agent.message` / `human.message` / `human.override` events

## Reproducibility

The renderer is pure projection — no provider calls, no mutation. Two
runs from the same snapshot produce identical markdown (modulo the
generation timestamp). The journal can be replayed through
`RoomProjector` to rebuild both the snapshot and the memory store from
scratch.
